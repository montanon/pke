# Roadmap

This roadmap lists possible future work. It is not part of the MVP commitment.

## Near-term

- improve nearby attestation reliability,
- add fallback transport,
- harden session lifecycle,
- improve duplicate and replay rejection,
- show exact signed payloads,
- visualize custody timeline,
- improve key grant UX,
- add user-selected metadata precision,
- add metadata preview before submission.

## Medium-term

### Public-chain anchoring

Periodically anchor ledger root hashes to a public chain. This should be optional and should not replace the custody ledger.

### Transparency-log model

Explore Merkle-tree or transparency-log designs with inclusion and consistency proofs.

### Organization workflows

Support organizations as recipients or reviewers, such as newsrooms, legal aid groups, human-rights organizations, and civic watchdogs.

### Stronger Sybil resistance

Explore organization-backed identities, device-bound credentials, limited reputation, verified witness groups, and privacy-preserving credentials.

### Hardware-backed security

Explore Secure Enclave-backed keys, hardware-backed identity, stronger local key protection, and attestation APIs where available.

## Long-term

- production trust-and-safety workflows,
- legal review and jurisdiction-specific compliance,
- legal evidence export formats,
- Android client,
- web verifier,
- command-line verifier,
- portable protocol core,
- public verification tooling,
- advanced proximity verification.

## Intentionally deferred

The following are not MVP priorities:

- tokens,
- proof of work,
- proof of stake,
- general-purpose blockchain,
- decentralized social network,
- public content feed,
- arbitrary encrypted storage,
- AI agent layer,
- automated content interpretation.

## Roadmap principle

Future features should strengthen verifiable chain of custody for private mobile evidence. Features that shift the product toward general social media, private file storage, or unsupported proof-of-truth claims should be avoided.
